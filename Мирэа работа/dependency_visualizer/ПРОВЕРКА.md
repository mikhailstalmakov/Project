# Инструкция по проверке в PowerShell

## Способ 1: Запуск через скрипт (самый простой)

1. Откройте PowerShell в VS Code (терминал уже открыт)
2. Перейдите в папку `dependency_visualizer`:
   ```powershell
   cd dependency_visualizer
   ```
3. Запустите скрипт:
   ```powershell
   .\test.ps1
   ```

Если возникнет ошибка о политике выполнения, выполните:
```powershell
Set-ExecutionPolicy -ExecutionPolicy RemoteSigned -Scope CurrentUser
```

## Способ 2: Ручной запуск команд

### Шаг 1: Перейдите в папку dependency_visualizer
В терминале PowerShell выполните:
```powershell
cd dependency_visualizer
```

### Шаг 2: Запустите тесты

**Тест 1: Базовый тест (Этапы 1, 2, 3, 5)**
```powershell
python main.py config_test.xml
```

**Тест 2: С фильтрацией**
```powershell
python main.py config_test_filter.xml
```

**Тест 3: Обратные зависимости (Этап 4)**
```powershell
python main.py config_test.xml --reverse E
```

**Тест 4: Реальный пакет из PyPI**
```powershell
python main.py config.xml
```

## Если команда `python` не работает

Попробуйте использовать `py`:
```powershell
py main.py config_test.xml
```

## Что должно произойти

При успешном запуске `python main.py config_test.xml` вы увидите:

```

  repo_url: test_repo.txt
  test_mode: True
  output_file: graph_test.svg
  filter_substring:

============================================================

============================================================

Total packages in graph: 26

```

## Проверка всех этапов

- ✅ **Этап 1**: Параметры конфигурации выводятся в начале
- ✅ **Этап 2**: Прямые зависимости выводятся в разделе "Stage 2"
- ✅ **Этап 3**: Полный граф строится, циклы обнаруживаются
- ✅ **Этап 4**: Используйте `--reverse E` для проверки
- ✅ **Этап 5**: Генерируется D2 код и SVG файл

## Результаты

После запуска будут созданы файлы:
- `graph_test.d2` - код на языке D2
- `graph_test.svg` - визуализация (если установлен D2)
